FAZA D · CATALOG + ASSET DETAIL · MOCKUP FLOW BOARD
FrameFlow — masonry catalog & the full asset-detail pattern

Static design board for the getframeflow.app Templates screen and the asset detail page, FAZA D — the final piece of the Artlist-style redesign. The catalog keeps everything above the grid (search hero · category chips · facet dropdowns · results + sort) and swaps the uniform grid for a masonry of FAZA C rich cards (two grid options below — pick one). The detail keeps the app's real pattern (a dedicated page, not a modal: breadcrumb + preview/meta split) and gains downloads/updated stats, a 6-cell spec grid, favorite + share actions, rich related cards, and a new “Part of collection” strip. Card aspect ratios come from data the catalog already has (t.ar per template). Nothing here touches the live platform.

D1 — what to look at

1
Above the grid nothing moves — search hero, category chips and the facet bar are the live A2 screen. The results + sort readout stays at the bar's right edge; the sort popover (shown open) is the standard va-fpop.
2
Masonry = data we already have. The live catalog already renders per-template aspect-ratio (t.ar) and has orientation facets — Option A just stops forcing rows. Landscape, vertical (9:16), square and 4:5 social cards sit in 4 CSS columns.
3
Rich card everywhere — same va-rc anatomy as Home (FAZA C): badges top-left, duration bottom-right, hover = scale 1.025 + play + heart/download reveal, meta line + category/downloads. One card system across the app.
4
Build note: CSS columns flows items down each column, so strict sort order reads column-first, not row-first. Real build either accepts that (Artlist does) or distributes items round-robin into N column stacks in JS to keep left-to-right rank. Decide at build time.
5
Reduced motion: reveal bar always visible, no scale, no play fade — identical behavior to the FAZA C cards.
D1b · ALTERNATIVE
Catalog — Option B · uniform rows with 2-wide feature cards

Same page, different grid: every card stays 16:9 in strict 4-column rows, and roughly every seventh template renders as a 2-wide feature card (editorial pick / top download). Calmer scan rhythm, perfect sort order preserved, vertical templates get letterboxed thumbs instead of tall cards. Pick A (true masonry — dynamic, media-forward, Artlist look) or B (uniform + feature — ordered, calmer). Everything above the grid is identical in both.

D1b — Option B rationale

1
Feature card = grid-column: span 2, bigger play affordance, meta gains an “Editor's pick” tail. Slot rule for the build: every 7th item, or a curated/featured flag from the API.
2
Trade-offs vs A: B keeps rows aligned and sort order exact, and never produces very tall cards; A shows every template in its true shape (verticals pop) and feels most like Artlist. B is also cheaper to build — plain CSS grid + span 2.
3
Vertical templates in B keep the 16:9 card and letterbox the preview (thumbnail already contains the vertical frame centered). If most social packs matter, that's a point for A.
D2 · DESKTOP 1280
Asset detail — the full pattern, enriched in place

The app's detail is a dedicated page (breadcrumb + preview/meta split — A2b), not a modal; FAZA D enriches it in place. New: a stats line (downloads · favorites · updated), the spec grid grows to six cells (+ orientation · format · downloads), the CTA row gains favorite + share, “More like this” thumbnails become full rich cards, and a new “Part of collection” strip anchors the page — the item shown inside its collection with a peek row and View collection.

D2 — what to look at

1
Same skeleton as live — breadcrumb → 1fr/384px split (preview left, meta right). Detail stays a page, deep-linkable; nothing turned into a modal.
2
Meta column enriched: stats line under the author (downloads · favorites · updated), spec grid grows from 4 → 6 cells (+ ORIENTATION · FORMAT · DOWNLOADS — replaces the duplicated AUTHOR cell). Data all exists in the catalog payload.
3
CTA hierarchy: Download pack keeps the spend-gradient pill (the one “money” action on the page); Open in After Effects stays ghost; favorite + share are new 44px ghost squares. For a PRO template on a Free plan, the primary swaps to “Get Pro to download” (same spend pill) and the gate note explains — same rule as today.
4
Related templates upgrade from bare thumbnails to full rich cards (badges, duration, hover reveal, meta). Source: same category + same app, top-downloaded, excluding self.
5
Part of collection — new strip: collection info + View collection at left, peek row at right with the current item outlined lime + “THIS ITEM”, then +N tile. Maps to the FAZA C shelves: collection = curated category slice; View collection deep-links to Templates with that filter. If an item belongs to no collection, the strip simply doesn't render.
D3 · MOBILE 390
Mobile — 2-column masonry catalog · stacked detail

Catalog: search hero compresses, chips + facets become horizontal scroll rows, the count/sort line sits above the grid, and the masonry collapses to 2 columns (reveal bar and hover motion are desktop-only — a tap opens the detail, same rule as FAZA C). Detail stacks: preview → title/meta → CTA row → specs → description → related (2-col) → collection card. Bottom tabs unchanged.

D3 — what to look at

1
Catalog: chips and facet chips are horizontal-scroll rows (same as live), count + sort collapse to one line above the grid, masonry runs at 2 columns with tighter gaps. Cards drop the hover reveal and category line — tap opens detail (FAZA C mobile rule).
2
Detail stacks in reading order: preview → badges/title/author → stats → CTA row (spend Download + heart + share) → plan gate → 2×2 specs → related (2-col rich cards) → collection card. The money action is above the fold right after the preview.
3
Collection card on mobile is a compact block: kick + name + count, 1-row peek (current item lime-outlined), full-width ghost View collection.
4
Breadcrumb truncates the item name; Templates/Category stay tappable for back-navigation.
