C1 — what to look at

1
Everything above “Featured models” is untouched — hero, credit card, quick actions and Jump back in are the live screen, byte-for-byte intent.
2
Featured models (Option A) — one spotlight “Model of the week” + a 2×2 rail. Real names/costs from FFAPI.models: Kling v3.0 ✦35/5s, Nano Banana Pro ✦8, Veo 3.1 Fast ✦20, Kokoro ✦8, SFX ✦3. “Try” = spend-gradient pill → opens AI Studio with that model pre-selected.
3
Richer card (rc-) replaces the old va-tc: 16/9 media, hover = 1.025 scale + play button + heart/download reveal, badges NEW (lime outline) / PRO (lime) / 4K, meta line gains category + download count. With prefers-reduced-motion the reveal bar is simply always visible and nothing moves.
4
Collection shelves (sh-) — horizontal scroll rows with right-edge fade + arrow, Artlist style. Real build derives them from the existing catalog: Trending = downloads/7d, Lower Thirds = category filter, New = createdAt. “View all” deep-links to Templates with that filter applied.
5
Reading order: your work (Jump back in) → what to try (models) → personalized picks → browse shelves. Monetization stays limited to “Add credits” + model “Try” pills.

C2 — A vs B, how to choose

A
Spotlight + rail (in C1): editorial, gives marketing a weekly “push” slot, big preview sells motion. Costs: the hero card dominates; rail cards carry less info (no description).
B
Uniform shelf (this crop): every model equal, each card fits name + one-liner + cost + Try. Scales to N models and matches the collection shelves visually. Less “wow”, more catalog.
→
Both read from FFAPI.models(mode); the featured set is a small curated list (config or “isFeatured” flag). Try = AI Studio deep-link with the model pre-selected — same handoff the Use ▾ menu already does.
