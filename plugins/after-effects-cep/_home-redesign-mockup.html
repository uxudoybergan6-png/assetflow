How it behaves

1#10 Home: the lime lightning block is pinned to the top-left of every screen — Catalog, every AI tool, history, settings, account. One tap = Home. On hover it glows; on Home itself it’s the logo.
2#11 Account: avatar pinned top-right on every screen, opens the account sheet. The credits pill next to it shows ✦ balance + plan (Free / Pro / Studio) and opens the same sheet on its Plan tab.
3Deep screens keep both anchors and add a local ‹ back. You can never get lost more than one tap from Home.
4Guest: the right cluster becomes one clear Sign-in action — same geometry, so nothing jumps after login.
✦Implementation note: this is the existing af-topbar / ai-hdr pattern unified — same heights, tokens and wiring (openAccountSheet, homeGo), so the build step is low-risk.

Why this direction works

APersonal, time-aware greeting (“Good evening, Dilnoza”) makes a utility panel feel like a studio, not a file browser. Name comes from the account; falls back to “Good evening — let’s make something move.”
BTwo pillar doorway cards answer the only question Home must answer: templates or AI? Blue tint = Catalog, lime tint = AI — matching the select/accent token roles already in the plugin.
CContinue rows resurface the last import (Re-import) and the last generation (Import into comp) with real thumbnails — the fastest path back into work.
DRecommended grid reuses the existing card language 1:1 (PRO/FREE badge, 4K chip, AE dot) — zero new card CSS at build time.
ECredits nudge translates the balance into outcomes (“~12 videos”) — gentle upsell without a paywall feel.
◱Above the fold (720px): greeting, both pillars and the first continue row — the resume action is visible without scrolling.

Why this direction works

ASearch-first: the field editors already look for sits on top, styled like the catalog toolbar (⌘K hint). Tapping it drops into the catalog with focus in the input — existing wiring.
BQuick-action tiles are the two pillars exploded into four one-tap verbs: browse, make an image, make a video, open downloads. Lime = generative, blue = library, consistent with token roles.
CFeatured hero carousel gives contributors’ best packs a stage; Import is available right on the card (frosted ghost pill on imagery — matches the built hm-feat pattern).
DJump back in is a horizontal rail mixing template imports, finished gens and even a draft prompt — resume anything, not just re-open it.
EAI spotlight banner is a slot for model launches / promos — future-proof space the backend can feed later.
◱Above the fold (720px): search, all four quick actions, the full hero and the start of “Jump back in” — everything actionable at first paint.

Onboarding logic

AWarm, specific promise (“Make After Effects move faster”) instead of a login form. The form itself stays in the existing account sheet (g2) — both CTAs open it; Google uses the shipped device-code flow.
BThree concrete value rows — templates, AI studio, sync — each mapped to a real shipped feature, no vaporware.
CCatalog peek: real cards, blurred, with a single lock chip. Browsing stays open (current behavior); only import is gated — the lock explains why to sign in.
DFree-plan facts in the footer set expectations early (3 imports/mo · ✦ 50) — fewer surprise paywalls later.
