Bir jumlada xulosa AssetFlow'ning backend va biznes-mantiqi ikkala referensdan ham kuchli; lekin UI/UX va dizayn izchilligi ortda. Yaxshi xabar — uchala plagin ham bir xil lime-yashil aksent oilasidan foydalanadi, shuning uchun ikkala dizaynni birlashtirib AssetFlow'ga qo'llash nafaqat mumkin, balki tabiiy. AI Tools esa mavjud auth + subscription tizimingiz ustiga proxy-route'lar orqali oson o'rnatiladi.

1. Boshqaruv xulosasi

Uchala plagin bitta janr — Adobe ichidagi kreativ kontent paneli. Ammo har biri boshqacha kuchli tomonga ega:

Higgsfield

React + Vite + Tailwind v4

Kuchli: Zamonaviy AI-tool UX. Model tanlash, prompt composer, parametr slayderlari, generatsiya progress effektlari — sanoat darajasida sayqallangan.

Dizayn tili: Qora fon + neon lime #d1fe17. Minimalist, ko'p bo'sh joy, yumaloq burchaklar, backdrop-blur.

Motion Factory

Vanilla JS + Web Components

Kuchli: Yig'iladigan yon-panel navigatsiyasi, Algolia qidiruv, hover-video kartalar, AI ovoz/SFX generatsiyasi to'liq integratsiya qilingan.

Dizayn tili: Charcoal #161515 + lime #91D010 + ko'k tanlash #327BFA. Inter shrifti, 200ms o'tishlar.

AssetFlow (sizniki)

Vanilla HTML/JS/CSS

Kuchli: To'liq SaaS backend, JWT+PluginToken auth, contributor workflow, moderatsiya, subscription, audit log, SSE.

Zaif: Dizayn izchilligi, juda mayda shriftlar, ikki xil aksent rang, hover-only tugmalar, AI bo'limi yo'q.

Asosiy xulosalar

  • ✓ Arxitektura ustunligi sizda — to'liq backend, contributor workflow va moderatsiya ikkala referensda ham yo'q.
  • ▲ Dizayn qarzi (design debt) — AssetFlow'da ikki avlod UI aralashgan: yashil brend bilan eski indigo #6366f1 qoldiqlari to'qnashadi; shriftlar 5–9px gacha tushib ketadi.
  • ✓ Ranglar allaqachon mos — Higgsfield #d1fe17, Motion Factory #91D010, AssetFlow #82c341 — uchchalasi bir oilada. Birlashtirish uchun rang almashtirish shart emas, faqat izchillashtirish kerak.
  • ✕ AI Tools yo'q — "creative-tools-saas" pozitsiyasiga ega mahsulot uchun bu eng katta funksional bo'shliq.

4. AssetFlow UI/UX kamchiliklari

Quyidagilar haqiqiy kod tahlilidan aniqlangan — ustuvorlik bo'yicha guruhlangan.

🔴 Yuqori ustuvorlik (foydalanuvchini to'g'ridan-to'g'ri xafa qiladi)

  1. O'qib bo'lmaydigan shriftlar. Chip 7px, featured strip 5–6px, meta 8–9px — HiDPI ekran va 380px panelda ko'rib bo'lmaydi. Yechim: minimum 11px, meta 10px, chip 9px.
  2. Ikki to'qnashuvchi aksent tizimi. Yashil #82c341 bilan eski indigo #6366f1/binafsha #a855f7 (featured banner, skeleton tugma, dropdown hover) aralashgan. Yechim: barcha indigo'ni o'chirish.
  3. Hover-only tugmalar. ★ sevimli, 🗑 o'chirish, overlay tugma — barchasi opacity:0, faqat hover'da ko'rinadi. Sichqonchasiz/birinchi qarashda topib bo'lmaydi. Yechim: doimo ko'rinadigan, kontrastli ikonalar.
  4. Yig'ilgan sidebar tooltipsiz. 380px'da nav 52px ikona-railga aylanadi, font-size:0 label, tooltip yo'q — 4 ikona noaniq. Yechim: hover-tooltip yoki Motion Factory uslubidagi yoyiluvchi sidebar.

🟡 O'rta ustuvorlik (izchillik va sayqal)

  1. Mos kelmaydigan radius va thumb nisbati. Karta 8px vs skeleton 12px; bitta elementda aspect-ratio:16/9 va 16/10 ikki marta — yuklanishda layout sakraydi.
  2. Browse va Admin umumiy dizayn tizimiga ega emas — turli CSS o'zgaruvchi nomlari, ikki xil mahsulotday ko'rinadi.
  3. Ortiqcha navigatsiya nazorati — scope <select> va sidebar nav bir xil holatni boshqaradi (chalkash).
  4. Soxta grid-view tugmasi ▦ — handler yo'q, mavjud bo'lmagan list-view'ni va'da qiladi.
  5. Profil bitta modalda ko'milgan — login, plan, download papka, contributor, admin — hammasi pastki account-sheet'da; alohida profil tab yoki sozlamalar ikonasi yo'q.
  6. Pagination / lazy scroll yo'q — har filtr/qidiruvda butun grid innerHTML orqali qayta chiziladi; katalog o'sgani sayin sekinlashadi.

🟢 Past ustuvorlik (yetuklik / a11y)

  1. A11y deyarli yo'q — custom dropdown'lar oddiy <div>, role/aria-expanded yo'q, klaviatura qo'llab-quvvatlash yo'q, Esc-yopish yo'q, focus-trap yo'q, toast uchun live-region yo'q.
  2. Light theme / theme toggle yo'q — prefers-color-scheme hisobga olinmaydi (AE'ning yorug' rejimida panel mos kelmaydi).
  3. Aralash til, i18n qatlami yo'q — nav inglizcha, qolgani o'zbekcha.
  4. Onboarding yo'q — yangi/login qilmagan foydalanuvchi bo'sh grid ko'radi, "ulanish" chaqiruvi yo'q.
  5. Katta preview / lightbox yo'q — faqat inline autoplay; pack ochmasdan kattaroq ko'rib bo'lmaydi.
  6. AI Tools / generatsiya bo'limi umuman yo'q (6-bo'limda hal qilamiz).
  7. Kolleksiya/papka, "yaqinda ko'rilgan", bulk amallar yo'q — faqat tekis Favorites + Downloaded.
  8. Per-karta yuklab olish indikatorlari yo'q — faqat bitta global progress bar.
  9. Server cold-start UI'da ko'rinadi ("server uyg'onmoqda ~60s") — UX hidi.

6. AI Tools bo'limini qo'shish

Savol: "AI tools bo'limi qo'shish uchun nimalar qilish kerak?"

AssetFlow'da allaqachon kuchli poydevor bor — auth (PluginToken), subscription (FREE/PRO), R2 storage, SSE. AI Tools shu poydevor ustiga proxy-route sifatida o'rnatiladi: API kalitlar serverda qoladi, kreditlar subscription orqali nazoratlanadi.

6.3 Backend: yangi route (apps/api/src/routes/ai.ts)

import { Router } from "express";
import { requirePluginAuth } from "../middleware/auth";
import { consumeAiCredits } from "../lib/plugin-profile";
import { uploadToR2, signUrl } from "../lib/storage";

const router = Router();

router.post("/voiceover", requirePluginAuth, async (req, res) => {
  const { text, voiceId, stability = 0.4 } = req.body;
  // ① kredit tekshirish (1000 belgi ≈ 28 kredit — MF formulasi)
  const cost = Math.ceil(text.length / 1000 * 28);
  const gate = await consumeAiCredits(req.user.id, cost);
  if (!gate.ok) return res.status(402).json({ error: "INSUFFICIENT_CREDITS" });

  // ② provayderga proxy (kalit faqat serverda)
  const audio = await fetch(`https://api.elevenlabs.io/v1/text-to-speech/${voiceId}`, {
    method: "POST",
    headers: { "xi-api-key": process.env.ELEVENLABS_KEY, "Content-Type": "application/json" },
    body: JSON.stringify({ text, voice_settings: { stability } }),
  }).then(r => r.arrayBuffer());

  // ③ R2'ga saqlash + signed URL (mavjud pattern)
  const key = `ai/voice/${req.user.id}/${Date.now()}.mp3`;
  await uploadToR2(key, Buffer.from(audio), "audio/mpeg");
  res.json({ url: await signUrl(key, 3600), creditsLeft: gate.remaining });
});

export default router;

// Generatsiya tugagach: faylni yuklab, AE'ga import qilish
async function importAiResult(signedUrl, fileName) {
  const tmpPath = await downloadToTemp(signedUrl, fileName);  // mavjud https.get pattern
  // mavjud host.jsx import funksiyasidan foydalanish
  evalJSX("importMediaFromPath", tmpPath);  // §5.2 typed wrapper
  showToast("AI natija AE'ga import qilindi ✓", "success");
}
