1. Boshqaruv xulosasi (texnik)

Savolingiz: "Texnik jihatdan ikkala raqibdan nimani olishim kerak, qaysi tizimlari menikidan kuchli va nega, qaysi bo'limlarni qo'shish kerak — to'liq strategiya." Quyida har bir tizim alohida tahlil qilingan.

⚡ ENG MUHIM STRATEGIK XULOSA — bitta harakat 3 muammoni hal qiladi AssetFlow plaginini Bolt CEP (Higgsfield ishlatadigan framework) ga ko'chirish bir vaqtning o'zida: (1) ZXP paketlash+imzolashni beradi (vite-cep-plugin ichida ZXP build bor — bu sizning №1 blokeringiz), (2) typed evalTS ko'prigini beradi (injection xavfini tugatadi), (3) zamonaviy komponentli UI beradi. Uchta eng katta texnik qarzni bitta migratsiya yopadi.

2. Tech-stack va arxitektura metodologiyasi

Har bir plagin tubdan boshqa metodologiyada qurilgan. Buni tushunish — nima ko'chirib bo'lishini aniqlaydi.

Higgsfield

Bolt CEP · Vite + React 19 + TS + Tailwind v4

Metodologiya: Yagona kompilyatsiya qilingan SPA. Vite bitta main-*.cjs (71k qator, minified) + init-cep-*.cjs ga to'playdi, source-map bilan.

ExtendScript: bitta jsx/index.js (2750 qator), ppro + aeft namespace.

Modul: index.html'da maxsus AMD/require shim.

State: TanStack React Query (server-state), evalTS bridge.

Motion Factory

Vanilla JS · 66 ta pixflow-* Web Component npm paket

Metodologiya: Mikro-modulli. Har funksiya alohida npm paket → customElements.define() bilan Web Component.

Kompozitsiya: main.js paketlarni require() qiladi; <mf-templates>, <mf-voiceover> kabi teglar.

Aloqa (3 kanal): CustomEvent + umumiy localStorage["init-data"] + window event-bus (enable-toolbar...).

Build: faqat SASS kompilyatsiya; JS xom.

AssetFlow (sizniki)

Monorepo · Express 5 + Prisma + Next + statik

Metodologiya: npm workspaces. apps/api (Express+TS), apps/web (Next 15), packages/database (Prisma 6.8), packages/assetflow-studio (statik HTML).

Plagin: plugins/after-effects-cep — build yo'q, xom HTML/JS/JSX. host.jsx 2229 qator, ~110 funksiya.

Deploy: API→Render, Studio→Vercel, lokal→PM2.

Ko'chirilishi bo'yicha xulosa Higgsfield'dan framework darajasidagi narsalarni olish mumkin (Bolt CEP, typed bridge, Query-cache, Sentry). Motion Factory'dan alohida tizimlarni olish mumkin (Algolia, auto-update, AI-proxy formulasi) — lekin uning xavfsizlik anti-naqshlarini emas. AssetFlow'ning monorepo backend'i ikkalasidan ustun — uni saqlang.

3. Tizim-bo'yicha chuqur taqqoslash

14 ta tizim. Har biri: HF qanday qiladi · MF qanday qiladi · siz qanday qilasiz · hukm + tavsiya.

① Build & paketlashRaqiblar kuchli
HiggsfieldBolt CEP = Vite. ZXP imzolangan paket. Source-map shipped. Hot-reload dev.
Motion Factorynpm + SASS build. ZXP imzolangan. 66 paket versiyalanadi.
AssetFlowBuild yo'q. Xom fayllar. ZXP yo'q — faqat install-cep.sh (imzosiz, dev-only).
Tavsiya: №1 blocker. Bolt CEP'ga ko'ching (vite-cep-plugin ZXP imzolashni o'z ichiga oladi) YOKI minimal yo'l: ZXPSignCmd + self-signed .p12 + scripts/build-zxp.sh. Bunsiz oxirgi foydalanuvchi o'rnata olmaydi.
② CEP ↔ ExtendScript ko'prigiHiggsfield kuchli
HiggsfieldTyped evalTS/listenTS/dispatchTS. Argumentlar JSON.stringify → try/catch ExtendScript → Promise. Namespace host["ai.higgsfield.cep"].fn().
Motion FactoryXom evalScript, qo'lda string concat. Fayl yo'llari to'g'ridan-to'g'ri interpolyatsiya → injection.
AssetFlowXom string: evalScript('importSingleSceneFromAep('+JSON.stringify(cfg)+')'). Typed wrapper yo'q. host.jsx evalScript limitidan oshib $.evalFile fallback ishlatadi.
Nega muhim: Xom string ko'prik = ① injection xavfi (zararli nom/yo'l ExtendScript bajaradi), ② type xavfsizligi yo'q, ③ har chaqiruvni qo'lda yozish — xatoga moyil. Higgsfield naqshi sanoat standarti.
Tavsiya: evalJSX(fn, ...args) typed wrapper qo'shing (Bolt CEP'ga ko'chsangiz tekin keladi). Argumentlarni faqat JSON sifatida uzating, ExtendScript ichida JSON.parse qiling — string concat qilmang.
③ Auth & sessiyaAssetFlow kuchli (1 kamchilik bilan)
HiggsfieldOAuth 2.0 + PKCE (S256) + Clerk. refresh_token bor. Token ~/.higgsfield-auth-*.json (fs). Bearer id_token.
Motion Factorydate-token ddmmyy (taxmin qilinadi). WebSocket login. localStorage'da token+email. Hardcoded pxAPIKey.
AssetFlowJWT (HS256, 7d) + PluginToken (32-bayt random, 30d, DB, server-validatsiya). prefs.json (disk). Refresh yo'q.
Olish (HF'dan): refresh_token flow — 30 kun tugaganda majburiy qayta-login o'rniga jim yangilash. Tuzatish: apply-ae-prefs hardcoded fayl yo'liga bog'liqligi (server+plagin bir mashinada deb faraz qiladi — faqat dev'da ishlaydi). OLMANG (MF'dan): date-token, localStorage auth, hardcoded key.
④ API dizayni & klientAralash
Higgsfieldfetch + TanStack Query (cache, polling refetchInterval, retry-lanes, abort-signal). Base fnf.higgsfield.ai.
Motion FactoryXom XMLHttpRequest, WordPress REST (23+ endpoint), wp-json/pixflow/*.
AssetFlowExpress 5, RESTful, Zod validatsiya, Prisma xato-mapping (P2025→404, P2002→409). Klient: xom fetch, har filtr/qidiruvda innerHTML qayta-chizish.
Server: sizniki kuchli (Zod + tipli xato-handling). Klient'da olish (HF'dan): cache + polling abstraktsiyasi (job-status uchun) va innerHTML teardown o'rniga incremental DOM yangilash. Vanilla'da ham kichik query-cache yozish mumkin.
⑤ QidiruvMotion Factory kuchli
HiggsfieldServer-side (galereya AI generatsiyaga qaratilgan, katalog qidiruv kam).
Motion FactoryAlgolia (appId WS11ARDO5G). Indekslar pixflow_elements+_trending. Typo-tolerant, optionalFilters, kategoriya-query AND-birikma. Klient-side, bir zumda.
AssetFlowfindMany + klient-side nav/cat filter. Relevance scoring qo'lda. Typo-tolerance yo'q, scale bo'lmaydi.
Nega muhim: Katalog o'sgani sayin findMany+JS-filter sekinlashadi, relevance/typo-tolerance bermaydi. MF 10k+ shablonni bir zumda qidiradi.
Tavsiya (bosqichma-bosqich): arzon → PostgreSQL tsvector/pg_trgm (sizda allaqachon Postgres bor). Keyin → Meilisearch/Typesense (self-host, arzon) yoki Algolia (bepul 10k so'rov/oy). Diqqat: MF'dek search-key'ni klientga hardcode qilmang — faqat search-only key, indeks faqat published yozuvlar.
⑥ Yuklab olish (download) quvuriAssetFlow kuchli
Higgsfieldfetch → fs.write → cache. os.tmpdir.
Motion FactoryPOST signed-url → https.get stream → progress (content-length) → AdmZip.extractAllTo → library API.
AssetFlowhttps.get qo'lda redirect (8 limit) → cross-origin'da Auth header'ni olib tashlaydi (token leak'dan saqlaydi) → shell unzip -o → .aep/.mogrt ajratish → selective scene download.
Sizniki eng xavfsiz — redirect'da Auth-header strip'i MF'da yo'q. Kichik yaxshilash: shell unzip o'rniga adm-zip (MF kabi) — system unzip mavjudligiga bog'liqlikni yo'qotadi (Windows'da yo'q).
⑦ Adobe'ga importAralash
HiggsfieldevalTS("importMediaFromPath", ...). Track Matte / nested sequence avtomatlashtirish.
Motion Factory5 app: PPRO importMGT/importFiles, AEFT ImportOptions, LUT ADBE Apply Color LUT2, PHXS/ILST open(File).
AssetFlowhost.jsx 110 funksiya — eng katta native sirt. Async open-project navbati (app.open() bloklanishini chetlab o'tadi), undo-group, item-ID diff bilan yangi comp topish.
Sizning host.jsx chuqurligi raqiblardan ustun (async navbat — aqlli yechim). Olish (MF'dan): ko'p-app host-detection naqshi (getHostEnvironment().appName switch) — kelajakda PPRO/PS qo'shsangiz. Tuzatish: evalScript limitini typed bridge + $.evalFile bilan to'g'ri boshqaring.
⑧ Fayl saqlash (storage)AssetFlow kuchli
HiggsfieldS3 pre-signed URL, 2-bosqichli upload (confirm).
Motion FactoryCDN to'g'ridan-to'g'ri (link share qilinishi mumkin).

Sizniki production-darajasi (OOM-fix, stream-upload). Bitta tuzatish: thumb/preview public R2 URL (URL bilan hammaga ochiq). Agar muhim bo'lsa — preview'ni ham signed qiling yoki ochiqligini ataylab qabul qiling.
⑨ Subscription / billing / kreditlarAssetFlow kuchli (1 bug bilan)

Sizning server-side gating'ingiz MF'dan tubdan xavfsiz. KRITIK BUG: PLUGIN_ALLOW_PRO_WITHOUT_STRIPE="true" production'da o'rnatilgan → hozir har kim PATCH /plan bilan bepul PRO oladi. Buni false qiling. Quirk: download oylik, import umrbod — assimetrik, hujjatlashtirib qo'ying.
⑩ Analytics & xato monitoringiHiggsfield kuchli

Motion FactoryGA4 (px-client). Discord webhook orqali xato hisoboti (leaked credential).

Nega muhim: Hozir production'da nima sinishini bilmaysiz — foydalanuvchi panel qotib qolsa, sizga signal kelmaydi. HF har xatoni stack-trace + breadcrumb bilan ko'radi.
Olish (HF'dan): ① Sentry (bepul 5k event/oy) — panel + API. ② Sentry'dan oldin sensitive-data filter regex (/(authorization|token|secret|password|api[_-]?key)/i) — token leak'dan saqlaydi. ③ Product analytics — Amplitude yoki PostHog (self-host).
⑪ Feature flagsHiggsfield kuchli

Olish (HF'dan): GrowthBook (bepul, self-host mumkin) yoki PostHog flags. Nega: AI funksiyalarni xavfsiz, foydalanuvchi-guruhiga rollout qilish; env-var o'rniga deploy'siz toggle.
⑫ Auto-updateMotion Factory kuchli
Higgsfieldmanifest.json polling → versiya solishtirish → brauzerda download link ochish.
Motion FactoryTo'liq avtomatik: GET get-plugin-latest-version-tag → download zip → rmdir + AdmZip.extractAllTo → o'zini almashtirish.

Nega muhim: Hozir tuzatish yoki yangi funksiyani foydalanuvchiga yetkaza olmaysiz — ular o'rnatilgan versiyada qotib qoladi.
Olish (MF'dan): versiya-tekshiruv endpoint (GET /api/plugin/version) + panel ochilishida solishtirish + notification (yoki MF'dek to'liq avtomatik download+extract). Tuzatish (MF bug'i): res.tag_name > current string solishtiruvi noto'g'ri ("1.10" < "1.9") — semver ishlating.
⑬ State boshqaruvi / realtimeAralash

Sizning SSE'ingiz raqiblardan yaxshi (ularda realtime yo'q). Diqqat: in-memory state Render restart'da yo'qoladi va horizontal-scale'da sinadi. Tavsiya: ko'p instansiyaga o'tganda Redis (rate-limit + SSE pub/sub + cache).

Siz MF'dan tubdan xavfsiz. Olib tashlash kerak: Admin paneldagi --disable-web-security (kerak emas bo'lsa). Olmang (MF): barcha 7 anti-naqsh. Tuzatish: evalScript injection (typed bridge bilan), Stripe-bypass flag, preview public-URL.

6. Nima qo'shish kerak va NEGA — modullar

Ustuvorlik tartibida. Har biri: nega (texnik sabab) + qaysi tech + qanday.

1 ZXP build + imzolash quvuri 🔴 BLOCKER
Nega: Imzolangan ZXP'siz oxirgi foydalanuvchi plaginni o'rnata olmaydi. Hozir faqat install-cep.sh (dev-only). Bu boshqa hamma narsadan oldin.
Qaysi tech: Bolt CEP (vite-cep-plugin ZXP o'z ichida) YOKI ZXPSignCmd + self-signed .p12.
Qanday: scripts/build-zxp.sh → sertifikat yaratish → ZXPSignCmd -sign ./dist app.zxp cert.p12 pass → tarqatish (yoki ZXP-installer).
2 Typed bridge (evalJSX) + ③ Sentry + ④ Refresh token
Nega: evalJSX — injection xavfini tugatadi va type-safety beradi. Sentry — production ko'rinishi (hozir 0). Refresh — 30 kun tugaganda majburiy login'ni yo'qotadi.
Qaysi tech: Bolt CEP evalTS (yoki qo'lda wrapper) · @sentry/browser + @sentry/node · JWT refresh endpoint.
Qanday: evalJSX(fn,...args) JSON-only · Sentry init + sensitive-filter regex · POST /api/plugin/refresh.
3 Qidiruv mexanizmi
Nega: findMany+JS-filter scale bo'lmaydi, typo-tolerance/relevance yo'q. Katalog o'sganda sekinlashadi.
Qaysi tech (bosqichma): ① PostgreSQL tsvector/pg_trgm (sizda Postgres bor — tekin) → ② Meilisearch/Typesense (self-host) → ③ Algolia (bepul 10k/oy).
Qanday: ContributorTemplate'ga searchVector ustun + GIN index + websearch_to_tsquery. Faqat APPROVED+published indekslanadi.
4 AI Tools bo'limi (yangi modul + daromad)
Nega: Mahsulot pozitsiyasi ("creative-tools"), raqobat pariteti, yangi daromad (kredit). Mavjud auth+subscription ustiga proxy sifatida o'rnatiladi.
Qaysi tech: ElevenLabs (ovoz/SFX) · fal.ai/Replicate (rasm) · OpenAI embeddings (semantik qidiruv) — hammasi server-side proxy.
Qanday: apps/api/src/routes/ai.ts → auth → kredit-gate → provider-proxy (kalit serverda) → R2 → signed URL. Prisma: AiGeneration + aiCredits. MF kabi kalitni klientga qo'ymang, exec(curl) ishlatmang.
5 Auto-update · ⑥ Feature flags · ⑦ Product analytics
Nega: Update — tuzatishni yetkazish. Flags — AI'ni xavfsiz rollout. Analytics — data-asosli qaror.
Qaysi tech: GET /api/plugin/version + semver · GrowthBook/PostHog flags · Amplitude/PostHog analytics.
8 Infratuzilma tuzatishlari
  • Stripe-bypass: PLUGIN_ALLOW_PRO_WITHOUT_STRIPE=false (hozir PRO bepul!).
  • Distributed state: ko'p-instansiyaga o'tганда Redis (rate-limit + SSE + cache) — hozir in-memory.
  • Dual asset model: Asset/Download (legacy) vs ContributorTemplate birlashtirish — maintenance qarzi.
  • Preview URL: public R2 — signed qilish yoki ataylab ochiq qabul qilish.
